В базе данных предлагается хранить одну таблицу - лог блокировок/разблокировок.
Информацию о клиенте можно узнать по UUID клиента, сопоставимого с UUID на сервисе, хранящем данные о клиентах.
Как вариант можно хранить здесь ИНН или ОГРН юрлица для большей читабельности
---
| Имя атрибута     | Тип данных    | Описание                                                         |
|------------------|--------------|------------------------------------------------------------------|
| id               | SERIAL / INT | Уникальный идентификатор записи                                  |
| client_id        | UUID / STRING| Уникальный идентификатор клиента. Сопоставим с UUID клиентов на сервисе, хранящем данные о клиентах |
| block_type       | STRING       | Тип причины блокировки (например, suspicion, details_mismatch). В будущем может расширяться  | 
| comment          | TEXT         | Дополнительный комментарий или причина блокировки               |
| blocked_at       | TIMESTAMP    | Дата и время установки блокировки. Если клиент незаблокирован - значение NULL                                |
| unblocked_at     | TIMESTAMP    | Дата и время снятия блокировки. Если клиент никогда не блокировался - значение NULL                                    |
| idempotency_key  | STRING       | Ключ идемпотентности для запросов блокировки/разблокировки                    |
